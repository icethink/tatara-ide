## 📋 Part 1: デフォルト設定（PHPStorm「最初にやれ」を全部入り）

### 1-1. エディタ基本設定

| 設定項目 | デフォルト値 | 理由 |
|---------|------------|------|
| フォント | JetBrains Mono 15px | 視認性最高のコーディングフォント。リガチャ対応 |
| テーマ | ダーク（Dracula系） | 目に優しい。PHPStorm 2024デフォルト準拠 |
| 行番号表示 | ON | コード指示・エラー特定に必須 |
| 空白文字の可視化 | ON（タブ・末尾スペース） | インデント問題の早期発見 |
| 現在行ハイライト | ON | カーソル位置の視認性向上 |
| ワードラップ | OFF（水平スクロール） | コード構造を崩さない |
| インデント | スペース4つ（PHP/JS/CSS共通） | PSR-12 / Laravel 標準 |
| 文字コード | UTF-8（BOM無し） | Laravel標準。Shift-JIS自動検出もON |
| 改行コード | LF（自動変換） | WSL/Linux標準。CRLF→LF自動変換 |
| 自動保存 | ON（フォーカス外した時） | ファイル保存忘れ防止 |
| 閉じタブ復元 | Ctrl+Shift+T | ブラウザと同じ操作感 |
| 配列キー整列 | ON | コードフォーマット時にkey=>value整列 |

### 1-2. PHP / Laravel 専用設定

| 設定項目 | デフォルト | 理由 |
|---------|----------|------|
| Laravel フレームワーク検出 | 自動ON | composer.json 検出で自動有効化 |
| Blade テンプレートサポート | 自動ON | @if, @foreach 等のハイライト・補完 |
| ide-helper 自動生成促し | プロジェクト初回に提案 | Facade/Model 補完の大幅強化 |
| コードスタイル | PSR-12 + Laravel Pint | Laravel公式スタイル |
| PHP バージョン指定 | composer.json から自動取得 | バージョン違いの構文エラー防止 |
| vendor ディレクトリ除外 | 自動除外 | 検索・インデックスから除外して高速化 |
| node_modules 除外 | 自動除外 | 同上 |
| .env ファイルハイライト | ON | 環境変数の視認性 |

### 1-3. JavaScript / Vue.js 設定

| 設定項目 | デフォルト | 理由 |
|---------|----------|------|
| ESLint 自動検出・実行 | ON | package.json 内の設定を自動検出 |
| Prettier 連携 | ON（保存時フォーマット） | コードスタイル統一 |
| Vue SFC (.vue) サポート | 自動ON | template/script/style の分割ハイライト |
| Emmet 展開 | ON（HTML/CSS/Blade内） | div.container>ul>li*5 で即展開 |
| TypeScript チェック | ON（tsconfig.json検出時） | 型エラーの早期発見 |
| Tailwind CSS 補完 | ON（tailwind.config.js検出時） | クラス名補完 |

### 1-4. 表示・外観のデフォルト

| 設定項目 | デフォルト | 理由 |
|---------|----------|------|
| レインボー括弧 | ON | 括弧の対応が一目瞭然 |
| パンくずリスト | ON（エディタ上部） | 現在のクラス/メソッド/スコープ位置表示 |
| ミニマップ | ON（右端） | ファイル全体の俯瞰 |
| インデントガイド | ON | ネストの深さを可視化 |

| エラー・警告の波線 | ON | リアルタイム問題検出 |

### 2-4. Laravel専用テンプレート・スニペット

```
# コントローラー
rc  → Resource Controller の雛形
ac  → API Controller の雛形

# モデル
mod → Model + fillable + casts の雛形
rel → hasMany / belongsTo リレーション生成

# マイグレーション
mig → Schema::create テーブル作成テンプレート
col → カラム追加スニペット（型選択式）

# Blade
@sec  → @section ... @endsection
@ext  → @extends('layouts.app')
@comp → <x-component /> テンプレート

# Route
rg → Route::get() テンプレート
rr → Route::resource() テンプレート
rm → Route::middleware() テンプレート

# テスト
tf → Feature Test テンプレート
tu → Unit Test テンプレート
```

### 3-7. プロジェクトテンプレート

作成後に自動で行うこと:
1. laravel new + 選択したオプション実行
2. .env のDB設定を自動反映
3. php artisan ide-helper:generate 実行
4. php artisan migrate 実行
5. npm install 実行
6. Git 初期コミット
7. プロジェクトをIDEで開く
8. artisan serve + npm run dev を自動起動
9. ブラウザでアプリを自動オープン

→ ボタン1つで「Hello World」が表示される状態まで全自動

## 📋 Part 4: 初心者がつまずく「あるある」と解決策

### 4-1. 環境系

| あるある | IDEでの解決策 |
|---------|-------------|
| 「artisan が動かない」 | ターミナルデフォルトをWSL bashに |
| 「composer not found」 | ウィザードで自動インストール+PATH設定 |
| 「npm run dev でエラー」 | バージョン自動チェック + nvm連携 |
| 「.env がない」 | 自動検出・コピー提案 |
| 「APP_KEY が空」 | 自動検出 → ワンクリックで生成 |
| 「permission denied」 | 自動検出 → ワンクリックで chmod 775 |
| 「WSLが遅い」 | 初回に警告 + 移動を提案 |
| 「Docker が起動しない」 | セットアップガイド表示 |
| 「ポートが使用中」 | ポート競合を検出 → 空きポート提案 |

### 4-2. コーディング系

| あるある | IDEでの解決策 |
|---------|-------------|
| 「クラスが見つからない」 | Alt+Enter で自動 import |
| 「メソッドが存在しない」 | ide-helper で自動解決 |
| 「ルートが404になる」 | Route一覧パネルで全ルート確認 |
| 「マイグレーションが失敗」 | migrate:status を視覚表示 + ロールバックボタン |
| 「Blade の変数が表示されない」 | 変数補完 + ヒント表示 |
| 「CSRF トークンエラー」 | フォーム作成時に @csrf を自動挿入提案 |
| 「N+1 クエリ問題」 | 警告表示（Debugbar連携） |

### 4-3. Git系

| あるある | IDEでの解決策 |
|---------|-------------|
| 「.env をpushしてしまった」 | .gitignore 自動生成 + 警告 |
| 「vendor/ をcommitしてしまった」 | 自動検出 → 除外提案 |
| 「コンフリクトの直し方がわからない」 | 3ペイン比較エディタ + 説明ガイド |
| 「git push が拒否される」 | pull → push のフローをGUIでガイド |
| 「ブランチの切替方法がわからない」 | ステータスバー常時表示 + 視覚的に切替 |
